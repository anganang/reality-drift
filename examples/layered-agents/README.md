# 目录级 path-scoped AGENTS.md 示例

演示规则怎么分层。同一套约束，跨模块通用的上移到根，模块特有的(协议、寄存器、时序、验证方式)留在模块目录。

```
your-project/
├── AGENTS.md              ← 根：所有模块都适用的规则（见 root-AGENTS.md）
└── src/
    └── radio/
        └── AGENTS.md      ← 模块：radio 特有的协议/时序/验证（见 radio-AGENTS.md）
```

为避免和仓库根的真实 `AGENTS.md` 混淆，这里两份示例分别命名为 `root-AGENTS.md` 和 `radio-AGENTS.md`，实际使用时去掉前缀放到对应目录。

## 判断条款放哪的标准

- 上移到根：单 owner、依赖单向、证据信任层级、单变量增量、提交纪律——换个模块同样成立。
- 留在模块：寄存器位定义、协议帧格式、具体时序窗口、这个模块的权威状态来源、这个模块怎么验证——只对本模块成立。

判断口诀:把这条规则搬到另一个模块还成立吗?成立就上移,不成立就留下。
